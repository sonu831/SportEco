## How Can I Contribute?

### Reporting Bugs

If you find a bug in the project, please check if it has already been reported by searching our [issues](https://github.com/sonu831/SportEco/issues). If it's not already there, please create a new issue and provide detailed information about the bug, including steps to reproduce it.

### Suggesting Enhancements

If you have ideas for enhancements or new features, feel free to open an issue and describe your proposal. We value your input and appreciate your suggestions.

### Code Contributions

We welcome code contributions! Here's how you can contribute:

1. Fork the repository to your GitHub account.
2. Clone your forked repository to your local machine.
3. Create a new branch for your feature or bug fix:
   ```sh
   git checkout -b feature-name
   ```
4. Make your changes and commit them with meaningful messages.
5. Push your changes to your forked repository:
   ```sh
   git push origin feature-name
   ```
6. Open a pull request to the main repository, explaining your changes and why they are valuable.

When you submit a pull request, make sure to:

- [x] Update the project version in the [package.json](package.json) file.
- [x] Mention contributors, e.g., "Yogendra Singh @Full Stack Developer has contributed to this project."
- [x] Use TypeScript for your code.
- [x] Follow the modular and component-based approach.
- [x] Format your code using Prettier.

Our team will review your pull request and provide feedback as needed. Thank you for your contribution!
